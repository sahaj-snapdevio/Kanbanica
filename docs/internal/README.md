# Internal / historical docs

These documents describe **how Kanbanica was originally built** — the phased
build plan, retroactive-change log, and an internal documentation audit. They
are kept for historical reference and are **not required reading** for
contributing.

For contributor-facing documentation, see:

- [ARCHITECTURE.md](../../ARCHITECTURE.md) — how the system fits together
- [CONTRIBUTING.md](../../CONTRIBUTING.md) — how to contribute
- the feature specs in [`docs/`](../) — task, sprint, permissions, real-time, etc.

Files here:

- `development-plan.md` — the original phase-by-phase build plan
- `revision-plan.md` — retroactive changes to already-built code
- `documentation-audit.md` — an internal review of the docs
