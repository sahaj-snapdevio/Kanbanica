### Orbit Admin Surface

Orbit is the admin UI at `/orbit` — same Next.js app, separate route group (`app/(orbit)/`) gated by the layout-level `requireAdmin()` redirect. No per-page auth checks needed at the page level (each server action / API route still re-checks admin).

**Sidebar groups** (`components/orbit/orbit-shell.tsx`) — _Overview_ (Dashboard), _Infrastructure_ (Regions, Servers, Storage, SSH Keys, Ports), _Management_ (Users, Spaces, Cubes, Snapshots, Domains, Plans, Subscriptions, Credit Purchases, Platform settings, Billing, Queues, Audit Logs).

**Detail-page layout convention — tabs.** Every Orbit entity detail page that has more than ~3 sections uses the shadcn `Tabs` primitive (`@/components/ui/tabs`) so the operator isn't scrolling one long view: **space-detail** (Overview · Members · Cubes · Plan · Subscription · Billing · Danger), **server-detail** (Overview · Cubes · Logs · Settings · Setup & Snapshots · Danger — auto-opens Setup for a not-yet-ready server via a `defaultTab` prop), **cube-detail** (Overview · Networking · Snapshots · Members · Activity, via the `CubeDetailTabs` shell, with the real-time Live-status card kept in a persistent right rail), and **user-detail** (Profile · Spaces · Danger). The page header + primary action ribbon stay pinned ABOVE the tabs (always reachable). Server components that need a tab shell pass their server-rendered cards into a thin `"use client"` shell as React-node slots (e.g. `ServerDetail`'s `setupSlot`/`snapshotsSlot`/`activitySlot`, `CubeDetailTabs`' `overview`/`networking`/… props) so no data fetching moves to the client. Reuse this pattern for any new detail page — do not reintroduce a single long scroll. The **space-detail** Cubes tab reuses the shared `CubesTable` (Sleep/Delete/Purge actions, scoped to that space's cubes); the **server-detail** Cubes tab keeps a read-only paginated table. **Active-tab state is URL-synced via the shared `useTabParam(validValues, fallback, paramKey="tab")` hook ([hooks/use-tab-param.ts](hooks/use-tab-param.ts)) — never `defaultValue` alone.** Spread it onto `<Tabs {...tabParam}>`; it reads the active tab from `?tab=` so a refresh / deep-link lands on the same tab, and writes the new value on switch with the native History API (`window.history.replaceState`, which Next keeps in sync with `useSearchParams`) so switching stays instant — NO `router.push`/`router.replace` (that would refetch the page's server component on every tab click). `server-detail` passes its `defaultTab` (e.g. `setup` for a not-yet-ready server) as the hook's `fallback`. The same hook backs the customer-facing tabbed pages too (e.g. the space **Members** page's Members/Invites tabs). The customer cube-detail uses route-based tabs (`<Link>` + `usePathname`, separate `/cubes/[id]/connect` etc. routes) and is already refresh-safe — leave it as-is.

**Dashboard** (`/orbit`) — 14-card grid of counts + links, plus a conditional "Unreported overage" card that renders only when the `polar.meter-reconcile` backlog is non-zero (`billing_events.overage_charge` rows where `polar_meter_reported_at IS NULL AND createdAt < now() - 5min`). The cube fleet is broken out into **Cubes** (total), **Running cubes**, **Sleeping cubes**, and **Error cubes** (each links to `/orbit/cubes?status=<value>`, which the client `CubesTable` pre-filters). Past-due subscriptions, pending top-ups, and error cubes switch their icon tile to amber + an amber border when their count is `> 0`. All counts query in parallel via one `Promise.all` (bounded `count()` queries, no scans).

**Subscriptions list** (`/orbit/subscriptions`) — every space with `providerSubscriptionId IS NOT NULL`, ordered by `currentPeriodEnd ASC NULLS LAST`. Columns: space (linked), owner email, plan (linked), status badge, MRR (= `plans.priceUsd`), period end (relative + tooltip), polar customer id (truncated + copy button), provider subscription id (truncated + copy button). Filter chips driven by `?status=` query param so the dashboard cards land pre-filtered; `past_due` chip is a meta-filter that includes both `past_due` and `unpaid` (both are "needs attention" states).

**Credit purchases list** (`/orbit/credit-purchases`) — last 200 `credit_purchases` rows, ordered DESC. Columns: created (relative + tooltip), space (linked), initiator email, base amount, processing fee, total, status badge, refunded amount, provider order id (truncated + copy button). Filter chips with `?status=` query param.

**Snapshots list** (`/orbit/snapshots`) — last 200 `cube_snapshots` rows. Columns: name, cube (linked), space (linked), status badge, size, backend, type (automatic / manual), created.

**Domains list** (`/orbit/domains`) — every `domain_mappings` row. Columns: domain, cube (linked), space (linked), server hostname, port, routing status badge, Cloudflare status badge, created. No mutations from this page — domain remove still lives on the cube detail page.

**Space detail subscription card** (`components/orbit/space-detail.tsx`) — Subscription card shows live state hydrated from `spaces` columns: plan (linked), status badge, period end + relative-time + 404-style "past" colorization, payment provider, polar customer id (copy), provider subscription id (copy), last subscription event timestamp, plus the existing Resync button. When the space has no provider subscription (`providerSubscriptionId IS NULL`), the card renders a single muted "This space is on the X plan with no provider subscription" line and hides the Resync button. Below the Subscription card, a **Recent plan-credit grants** card renders the last 5 `subscription_credit_grants` rows for the space (plan / period / amount / reason / granted-at).

**Spaces table** (`components/orbit/spaces-table.tsx`) — extended with `Plan` (linked to plan detail) and `Subscription` (status badge or `—` for no-subscription) columns alongside name / owner / credits / cubes / created.
