# Security Policy

## Supported versions

Kanbanica is pre-1.0 and under active development. Security fixes are applied to
the latest `main` branch. Once `v1.0.0` is released, this table will track
supported release lines.

| Version | Supported |
|---------|-----------|
| `main` (latest) | ✅ |
| < 1.0 tagged releases | ⚠️ best effort |

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Instead, report it privately through one of these channels:

1. **GitHub Private Vulnerability Reporting** (preferred) — open the repository's
   **Security → Report a vulnerability** tab to file a private advisory.
2. **A dedicated security contact** — if this deployment/fork publishes a
   security contact address, use it.

> **Maintainers / self-hosters:** before publishing your repository, enable
> GitHub's Private Vulnerability Reporting (Settings → Security → Private
> vulnerability reporting) and/or add your own monitored security contact
> address here. Do not ship this file pointing at an unmonitored inbox.

Please include:

- A description of the vulnerability and its impact.
- Steps to reproduce (proof-of-concept if possible).
- Affected version / commit and any relevant configuration.

## What to expect

- We aim to **acknowledge** your report within a few business days.
- We'll investigate, keep you updated on progress, and coordinate a fix and
  disclosure timeline with you.
- We'll credit you in the release notes unless you prefer to remain anonymous.

Thank you for helping keep Kanbanica and its users safe.
